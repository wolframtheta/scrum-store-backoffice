# Gestió d'Entorns

Aquest projecte utilitza fitxers `.env` per gestionar les configuracions d'entorn.

## Fitxers d'Entorn

Els fitxers d'entorn es troben a l'arrel del projecte:

- `.env.local` - Desenvolupament local
- `.env.pre` - Pre-producció / Staging
- `.env.pro` - Producció

### Format dels Fitxers `.env`

```env
PRODUCTION=false|true
API_URL=https://api.example.com/api/v1
```

## Com Funciona

1. Abans de servir o compilar l'aplicació, s'executa l'script `scripts/set-env.js`
2. Aquest script llegeix el fitxer `.env.*` corresponent
3. Genera automàticament el fitxer `src/environments/environment.ts`
4. Angular utilitza aquest fitxer generat durant la compilació

**Important**: El fitxer `environment.ts` es genera automàticament i NO s'ha de modificar manualment ni cometre a git.

## Scripts Disponibles

### Desenvolupament

```bash
# Servir en entorn local (port 4300)
npm run start
npm start

# Servir en entorn pre-producció
npm run start:pre

# Servir en entorn producció
npm run start:pro
```

### Compilació

```bash
# Compilar per local
npm run build

# Compilar per pre-producció
npm run build:pre

# Compilar per producció
npm run build:pro
```

## Afegir Noves Variables d'Entorn

1. Afegeix la variable als fitxers `.env.*` que necessitis:
   ```env
   NEW_VARIABLE=valor
   ```

2. Modifica `scripts/set-env.js` per llegir i afegir la nova variable:
   ```js
   const newVariable = envConfig.NEW_VARIABLE || 'default';
   ```

3. Actualitza el template de generació en `set-env.js`:
   ```js
   const content = `// Auto-generated by set-env.js
   export const environment = {
     production: ${production},
     apiUrl: '${apiUrl}',
     newVariable: '${newVariable}'
   };
   `;
   ```

4. Utilitza la nova variable al codi:
   ```typescript
   import { environment } from '@env/environment';
   console.log(environment.newVariable);
   ```

## Troubleshooting

### Error: Environment file not found

Si veus aquest error, assegura't que existeix el fitxer `.env.{entorn}` corresponent a l'arrel del projecte.

### La configuració no s'actualitza

Si canvies un fitxer `.env` mentre el servidor està executant-se, hauràs de:
1. Aturar el servidor (Ctrl+C)
2. Tornar a executar el comando corresponent (`npm start`, etc.)

Això regenerarà l'`environment.ts` amb els nous valors.
