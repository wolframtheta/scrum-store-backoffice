# Gestió d'Entorns

Aquest projecte utilitza fitxers `.env` per gestionar les configuracions d'entorn en **desenvolupament local**, i **variables d'entorn** per a **Docker/Coolify**.

## Fitxers d'Entorn (Desenvolupament Local)

Els fitxers d'entorn es troben a l'arrel del projecte:

- `.env.local` - Desenvolupament local
- `.env.pre` - Pre-producció / Staging
- `.env.pro` - Producció

### Format dels Fitxers `.env`

```env
PRODUCTION=false|true
API_URL=https://api.example.com/api/v1
```

**Important**: Aquests fitxers NO es pugen a git (estan al `.gitignore`) per seguretat.

## Com Funciona

### Desenvolupament Local

1. Abans de servir o compilar l'aplicació, s'executa l'script `scripts/set-env.js`
2. Aquest script llegeix el fitxer `.env.*` corresponent
3. Genera automàticament el fitxer `src/environments/environment.ts`
4. Angular utilitza aquest fitxer generat durant la compilació

### Coolify / Docker

1. L'script `set-env.js` detecta que no existeixen els fitxers `.env`
2. Llegeix les variables d'entorn del sistema (`process.env`)
3. Genera el fitxer `environment.ts` automàticament
4. El build continua normalment

**Important**: El fitxer `environment.ts` es genera automàticament i NO s'ha de modificar manualment ni cometre a git.

## Scripts Disponibles

## Configuració a Coolify

A Coolify, configura les següents variables d'entorn per cada projecte:

### Per Producció

```env
PRODUCTION=true
API_URL=https://store.scrum-app.com/api/v1
```

### Per Pre-producció

```env
PRODUCTION=true
API_URL=https://store.scrum-app.com/api/v1
```

### On configurar-les a Coolify

1. Ves al teu projecte a Coolify
2. Navega a **Environment Variables**
3. Afegeix les variables `PRODUCTION` i `API_URL`
4. Guarda i re-desplega

L'script de build automàticament detectarà aquestes variables i generarà la configuració correcta.

## Afegir Noves Variables d'Entorn

1. **Desenvolupament Local**: Afegeix la variable als fitxers `.env.*`:
   ```env
   NEW_VARIABLE=valor
   ```

2. **Coolify**: Afegeix la variable a les variables d'entorn del projecte

3. Modifica `scripts/set-env.js` per llegir la nova variable:
   ```js
   const newVariable = envConfig.NEW_VARIABLE || 'default';
   ```

4. Actualitza el template de generació:
   ```js
   const content = `// Auto-generated by set-env.js
   export const environment = {
     production: ${production},
     apiUrl: '${apiUrl}',
     newVariable: '${newVariable}'
   };
   `;
   ```

5. Utilitza la nova variable al codi:
   ```typescript
   import { environment } from '@env/environment';
   console.log(environment.newVariable);
   ```

## Troubleshooting

### Error: Environment file not found (desenvolupament local)

Assegura't que existeix el fitxer `.env.{entorn}` corresponent a l'arrel del projecte:
- Pots crear-los a partir de `.env.example`
- `cp .env.example .env.local`

### La configuració no s'actualitza

**Desenvolupament local**: Si canvies un fitxer `.env`:
1. Atura el servidor (Ctrl+C)
2. Torna a executar el comando (`npm start`, etc.)

**Coolify**: Si canvies les variables d'entorn:
1. Guarda els canvis a Coolify
2. Re-desplega l'aplicació

### Build falla a Coolify

Verifica que has configurat totes les variables d'entorn necessàries:
- `PRODUCTION`
- `API_URL`

L'script automàticament utilitzarà aquestes variables si no troba els fitxers `.env`.
